# 26_CI_PAGE — CI 페이지 진흥원 구조 이식 (단독)

진흥원(gidp.kr/gidp/gidp/gidp_ci)과 동일한 섹션 순서·레이아웃으로 /about/ci를 구성한다. 텍스트는 우리 학과용 플레이스홀더로 넣되(사용자가 어드민에서 수정), 구조·순서·다운로드 버튼은 진흥원과 동일하게. 우리 디자인 토큰(다크·글래스·radius 4) 적용, 진흥원 red/charcoal 색상값은 우리 것으로 대체하지 말고 "전용색상" 섹션은 편집 가능한 색상 슬롯으로 비워둔다.

## 섹션 순서 (진흥원과 동일)
1. 페이지 배너: "CI" (브레드크럼 About > CI)
2. CI의 의미: 대표 심벌 이미지 슬롯 + 설명 텍스트(편집) + 다운로드 3버튼(매뉴얼/JPG/AI)
3. 구성요소별 의미: 곡선 / 컬러 / 워드마크 (각 소제목 + 설명 텍스트 편집 + 필요시 이미지 슬롯)
4. 로고가이드: 한글 타입(국문 가로형 이미지 슬롯) / 영문 타입(영문 가로형 이미지 슬롯)
5. 시그니처: 상하조합형 / 좌우조합형 (각 이미지 슬롯)
6. 전용색상: Main Color / Secondary Color (색상 칩 + 이름 + 값, 편집 가능한 색상 슬롯. 지금은 비워두거나 예시 슬롯만)
7. 그래픽모티브: 이미지 슬롯

## 다운로드 버튼 (CI의 의미 섹션 내)
3개 버튼: "매뉴얼 다운로드", "JPG 다운로드", "AI 다운로드". 각각 아래 정적 파일을 다운로드(a download 속성). 파일 부재 시 버튼 비활성 또는 "준비 중" 상태.
- 매뉴얼(PDF): /ci/dah-ci-manual.pdf
- JPG: /ci/dah-ci.jpg
- AI: /ci/dah-ci.ai

## 이미지 슬롯 경로 (사용자가 나중에 파일 넣음, ImageFrame 사용)
- 대표 심벌: /ci/symbol.png
- 한글 가로형: /ci/logo-kr.png
- 영문 가로형: /ci/logo-en.png
- 시그니처 상하: /ci/signature-vertical.png
- 시그니처 좌우: /ci/signature-horizontal.png
- 그래픽모티브: /ci/motif.png
파일 없으면 ImageFrame 플레이스홀더("이미지 준비 중") 표시. client/public/ci/ 폴더를 생성.

## 편집
전 섹션 텍스트와 색상 칩(이름·값)을 어드민에서 수정 가능(코드쉐어링·나노디그리와 동일한 단일 문서 편집 패턴). 다운로드 파일과 이미지는 어드민 업로드 또는 파일 교체 둘 다 지원. ci 단일 문서 스키마에 각 섹션 필드 반영.
