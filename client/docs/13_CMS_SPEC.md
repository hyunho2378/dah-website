# 13_CMS_SPEC — 코딩 없는 유지보수 시스템

목표(사용자 정의 그대로): 코딩하는 사람과 안 하는 사람이 공존하며 유지보수되는 것이 진짜 AX. 디자인은 한 번 잠그고, 이후엔 "게시물 추가" 버튼만으로 사이트가 자란다.

## 2. 리치 에디터 (UX Study 사이트 기능 이관)

- 기반: contenteditable 자체 구현 대신 Tiptap(ProseMirror, MIT) 채택 권고 — 자체 구현은 리스크 과대. JSON(body jsonb) 저장
- 지원(스터디 사이트 패리티): 볼드·이탤릭·밑줄, H2·H3, 번호·불릿 리스트, 줄간격, 인용, 구분선, 링크, 이미지(Blob), 표
- 임베드 3종: 유튜브 / 피그마 / 구글 슬라이드 — URL 붙여넣기 시 자동 글래스 프레임 임베드
- 렌더러: body JSON → React 컴포넌트 매핑(우리 토큰으로만 스타일). 에디터 CSS가 프론트로 새는 것 금지
- 붙여넣기 클린업: 워드·구글독스 붙여넣기 시 스타일 스트립

## 3. 템플릿 잠금 원칙

작성자는 내용만 넣는다. 레이아웃·폰트·색은 템플릿이 강제한다. 에디터에 색상 피커·폰트 선택 기능 자체를 넣지 않는다. 이것이 "화려한 디자인을 잡아놓고 글쓰기처럼 추가"의 구현 방식이다.

## 4. 공유 버튼 (전 상세 페이지, 비로그인 포함)

- 다이나믹 아일랜드 패턴: 필 탭 → 확장(링크 복사 / X / 카카오는 URL 스킴만) 
- 링크 복사: navigator.clipboard, 성공 시 필 안에서 "복사됨" 상태 1.5초
- OG 메타: 상세 페이지별 동적 title·description·이미지(포스터/메인 이미지)

## 5. i18n (KR/EN)

- 구조: /en/* 미러 라우트 + LangContext. UI 라벨은 src/i18n/ko.js·en.js 사전
- 콘텐츠: title_ko·title_en, body는 ko 필수·en 선택. en 부재 시 영문 페이지에서 "Korean only" 뱃지와 국문 렌더(기계번역 자동 삽입 금지)
- 품질 기준: 대학원 랩 수준. 고정 페이지(About·교육과정·코드쉐어링) 영문은 사람이 감수한 원고를 사전 파일로 관리. Track 영문명: Design Track / AI Track / Enter-Culture Track
- 헤더 EN 토글은 KPC 위치(우측 유틸) 이식

## 6. 어드민 대시보드 (/admin)

- 홈: 콘텐츠 유형별 카운트 + pending 쇼케이스 큐 + 접수 현황(기간 중)
- 유형별 리스트 → 작성·수정 폼(위 매트릭스)
- 사용자 관리(owner): 이메일·롤 등록, 리셋
- 설정: 히어로 버튼, 접수 노출, 전시회 일정
- 내보내기(owner): 스냅샷 JSON 다운로드 + 서버가 파일 기록
- 어드민 UI도 동일 디자인 시스템(우주+글래스). 별도 관리자 테마 금지
