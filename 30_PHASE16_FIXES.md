# 30_PHASE16_FIXES — 크리틱 배치 (Opus, 병렬 2)

원문 변경 금지. Task로 U1(공개 UI·영상)·U2(어드민·데이터·모달) 병렬 후 통합.

═══════════ AGENT-U1: 공개 UI + 히어로 영상 ═══════════

## U1-1. 학생 성과 수상자 이름 강조
학생 성과 본문(국문·영문 둘 다)에서 사람 이름을 화이트(text.pri) + 볼드로 강조하라. 본문 원문은 변경하지 말고, 렌더 시 이름에 해당하는 부분만 스타일 적용. 방법: 본문에서 "학생이", 쉼표로 나열된 인명 패턴을 인식하거나, 수상자 필드(U2-2에서 추가)가 있으면 그 이름들을 본문에서 매칭해 강조. 과잉 매칭(일반 단어까지 볼드) 금지, 인명만.

## U1-2. 공모전·특강 레이아웃을 전시회와 동일 크기
공모전·특강 목록의 카드·그리드를 전시회 일반 섹션(상단 피처드 제외한 그리드 큐레이션)과 동일한 크기·비율·간격으로 통일하라. 현재 크기가 다르면 전시회 그리드 기준에 맞춘다. 포스터 2:3 ImageFrame, radius 4, 동일 카드 패딩.

## U1-3. 히어로 영상 즉시 로드
히어로 영상이 용량 대비 늦게 뜬다. 즉시 표시되게:
1. video 태그에 preload="auto", playsInline, muted, loop, autoplay. poster 속성에 정지 프레임 이미지(public/videos/hero-poster.jpg, 없으면 첫 프레임 캡처 대체)를 지정해 로딩 중에도 화면이 비지 않게.
2. index.html head에 <link rel="preload" as="video" href="/videos/hero.mp4">로 우선 로드.
3. 영상 로드 전까지 poster 또는 cosmos 배경이 보이고, 로드되면 부드럽게 영상으로 전환(깜빡임 없이).
4. 모바일에서는 영상이 화면에 맞게 축소·크롭 유지.

═══════════ AGENT-U2: 어드민 + 데이터 + 모달 ═══════════

## U2-1. 로그아웃 모달 z-index·딤 [실측 원인 확정]
실측 결과 헤더가 z-50인데 로그아웃 확인 모달이 그보다 낮아 헤더·콘텐츠에 겹쳐 최하단에 깔려 보인다. 수정:
1. 모달 오버레이(딤 배경)와 모달 패널을 z-index 최상위(예 z-[100] 이상, 헤더 z-50보다 확실히 위)로.
2. 딤 배경: 전체 화면 fixed inset-0, 검정 반투명(rgba 0,0,0,0.6), 모달 뒤를 어둡게.
3. 모달 패널은 딤 위에 중앙 정렬, 맨 앞. body 스크롤 잠금.
4. 다른 모달(로그인·공유 등)도 같은 z-index 체계로 통일해 겹침 방지.

## U2-2. 학생 성과 입력 필수 필드 강화
학생 성과 작성·수정 폼에서:
1. 영어 제목뿐 아니라 영어 본문도 필수 입력(둘 다 비면 발행 불가).
2. "수상자" 필드를 신설하고 필수로. 수상자 미입력 시 발행 불가.
3. 발행(publish) 전 검증: 국문 제목·국문 본문·영문 제목·영문 본문·수상자가 모두 채워져야 발행 가능. 하나라도 비면 발행 버튼 비활성 + 어떤 필드가 비었는지 안내.

## U2-3. (U1-1 연동) 수상자 데이터
achievement 스키마에 수상자(awardees) 필드 추가. 기존 데이터는 본문에서 추출하거나 비워두되, 신규·수정부터 필수. U1-1의 이름 강조가 이 필드를 활용할 수 있게.
